# CargoImport

Generated by `import_cargo_data.py` — re-run any time game data updates.

## cargos/
- `catalog.json` — every cargo row + payment / volume / type fields. Names from this list are the keys for `inputs` / `outputs` cargo maps in `delivery_points.json`.
- `types.txt` — distinct `EDeliveryCargoType` values. ONLY these are valid in `input_types` / `output_types`.
- `cargo_names.txt` — flat alphabetical list of cargo Names.

## delivery_points/
One `<ClassName>.example.json` per vanilla delivery-point BP. Each file is a
ready-to-paste `delivery_points.json` entry — drop it under any KEY (the
scene placeholder is `DeliveryPoint_<KEY>`) and adjust.

`visuals_seen` shows whatever marker/color/icon-shaped fields the CDO
actually exposes — use it to identify which knobs the framework can
expose next.
